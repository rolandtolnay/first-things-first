<task type="auto">
  <name>Task 2: Configure Tailwind with CSS variables for theming</name>
  <files>tailwind.config.ts, src/app/globals.css</files>
  <action>
    Update tailwind.config.ts to use CSS custom properties for theme colors. This enables easy theme switching.

    Define semantic color variables in globals.css:
    - --background, --foreground (base colors)
    - --card, --card-foreground (component backgrounds)
    - --primary, --primary-foreground (teal/cyan accent from JARVIS aesthetic)
    - --muted, --muted-foreground (subtle elements)
    - --border, --input (form elements)
    - --ring (focus states)
    - --destructive (error states)

    Create :root (light mode) and .dark (dark mode) variants.

    For JARVIS aesthetic, use teal/cyan as primary:
    - Light mode: subtle teals on white
    - Dark mode: glowing teals on dark background (#0a0a0f or similar)

    Configure Tailwind to reference these CSS variables so classes like bg-background, text-foreground work.
  </action>
  <verify>npm run build passes with updated config</verify>
  <done>CSS variables defined for both themes, Tailwind configured to use them</done>
